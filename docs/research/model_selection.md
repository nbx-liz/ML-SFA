# ML-SFA モデル選定資料

**作成日:** 2026-03-22

---

## 1. SFA分野の課題マップ

従来のパラメトリックSFAが抱える課題と、ML手法がどの課題にアプローチするかを整理する。

### 1.1 課題の分類

```
SFAの課題
├── C1: フロンティア関数形への依存
│     Cobb-Douglas vs translog で効率性推定値が大きく変わる
│     → ML手法の主要ターゲット（ノンパラメトリック化）
│
├── C2: 非効率性分布仮定への感度
│     half-normal vs truncated-normal で効率性ランキングが変動
│     → ベイズ手法・ノンパラメトリック分布で対処
│
├── C3: 誤差分解の識別困難性
│     v と u の分離が分布仮定のみに依存（脆弱）
│     → SFA固有の根本問題。ML単独では解決困難
│
├── C4: 非線形関係の捕捉不足
│     投入・産出間の複雑なパターンを捉えられない
│     → NN / GP / ツリーの得意領域
│
├── C5: 経済学的解釈性の欠如（ML導入時）
│     MLのブラックボックス性。単調性・凹性の保証なし
│     → Shape制約、構造化アーキテクチャで対処
│
└── C6: パネルデータ・異質性への対応
      企業間異質性、時変効率性の扱い
      → パネルNN、潜在群構造で対処
```

### 1.2 課題の重要度と難易度

| 課題 | 重要度 | ML介入の効果 | 難易度 |
|------|--------|-------------|--------|
| C1: 関数形依存 | 高 | **高** — ML の直接的な貢献 | 中 |
| C2: 分布仮定感度 | 高 | 中 — ベイズ手法で緩和可能 | 高 |
| C3: 識別困難性 | 最高 | **低** — ML単独では解決困難 | 最高 |
| C4: 非線形関係 | 中 | **高** — ML の得意領域 | 低〜中 |
| C5: 解釈性 | 中 | 中 — 制約付きMLで対処 | 中 |
| C6: パネル対応 | 中 | 中 — パネル構造の活用 | 高 |

**→ ML 介入の効果が最も高いのは C1（関数形依存）と C4（非線形関係）。**
これらを主軸に据えるのが合理的。

## 2. 候補モデルの PRO/CON 比較

### 2.1 ParametricSFA（従来型 — ベースライン）

| 項目 | 内容 |
|------|------|
| **概要** | Cobb-Douglas / translog + MLE + Jondrow条件付き期待値 |
| **代表文献** | Aigner et al. (1977), Jondrow et al. (1982) |
| **対象課題** | —（ベンチマーク基準） |

| PRO | CON |
|-----|-----|
| 理論が確立されており解釈が明確 | 関数形に強く依存 (C1) |
| 閉じた形の尤度で高速計算 | 非線形関係を捉えられない (C4) |
| 少量データでも安定推定 | 分布仮定への感度 (C2) |
| 全MLモデルの比較基準として必須 | |
| 実装難易度が低い | |

**実装工数:** 小（1-2週間）
**研究上の新規性:** なし（ただし必須の基盤）

---

### 2.2 NNFrontierSFA（ニューラルネットワーク型）

3つのアプローチが存在する。

#### A. 2段階NN-SFA

| 項目 | 内容 |
|------|------|
| **概要** | Stage 1: MLP でフロンティア推定 → Stage 2: 残差にパラメトリック分解 |
| **代表文献** | Kutlu (2024), Pendharkar (2023) |
| **対象課題** | C1, C4 |

| PRO | CON |
|-----|-----|
| 実装が比較的シンプル（各stageが独立） | Stage 1 が非効率性 u を吸収するリスク |
| 既存SFA推定コードを Stage 2 で再利用可能 | generated-regressor 問題 |
| フロンティアの柔軟性が高い | 2段階の理論的正当化が弱い |
| PyTorch + scipy の組合せで実装可能 | 単調性制約の実装が追加で必要 |

**実装工数:** 中（2-3週間）
**研究上の新規性:** 中

#### B. 同時推定NN-SFA

| 項目 | 内容 |
|------|------|
| **概要** | SFA対数尤度をカスタム損失関数に埋め込み、フロンティアと誤差分解を同時最適化 |
| **代表文献** | Tsionas, Parmeter, Zelenyuk (2023) |
| **対象課題** | C1, C2, C3, C4 |

| PRO | CON |
|-----|-----|
| 理論的に最も整合的（同時推定） | 最適化が困難（非凸、局所最適） |
| 2段階法の u 吸収問題を回避 | ハイパーパラメータが多い |
| フロンティアと分布パラメータを共学習 | 収束の保証がない |
| 研究の最前線で注目度が高い | 実装・デバッグの難易度が高い |

**実装工数:** 大（3-5週間）
**研究上の新規性:** 高

#### C. 構造化NN (xNN-SF)

| 項目 | 内容 |
|------|------|
| **概要** | SFA構造をNNアーキテクチャに反映。3サブネット（フロンティア/非効率性/ノイズ） |
| **代表文献** | Zhao (2024) |
| **対象課題** | C1, C4, C5 |

| PRO | CON |
|-----|-----|
| 解釈性とML柔軟性の両立 | 2024年提案で実績が少ない |
| 単調性をアーキテクチャで保証 | アーキテクチャ制約が表現力を制限する可能性 |
| SFA分解が構造的に組み込まれる | 独自アーキテクチャの設計・検証コスト |

**実装工数:** 大（3-4週間）
**研究上の新規性:** 高

---

### 2.3 GPFrontierSFA（ガウス過程型）

| 項目 | 内容 |
|------|------|
| **概要** | GPyTorch でフロンティアを推定 + カスタムSFA尤度 |
| **代表文献** | 直接の先行研究が少ない（開放的研究領域） |
| **対象課題** | C1, C2, C4 |

| PRO | CON |
|-----|-----|
| 不確実性定量化が自然に得られる | 計算量 O(N³)、大規模データに不向き |
| カーネル選択でフロンティアの滑らかさを制御 | 片側誤差分解のGPフレームワーク内実装が未確立 |
| ベイズ的枠組みで過適合に強い | 先行研究が少なく理論的裏付けが薄い |
| GPyTorch の変分推論でスケーラビリティ改善可能 | 高次元入力で性能劣化（次元の呪い） |
| **研究としての新規性が最も高い** | 技術的リスクが最も高い |

**実装工数:** 大（3-5週間）
**研究上の新規性:** 最高（先行実装がほぼない）

---

### 2.4 KernelSFA（カーネル型）

| 項目 | 内容 |
|------|------|
| **概要** | 局所多項式回帰 + 局所MLE、またはNadaraya-Watson推定 |
| **代表文献** | Fan et al. (1996), Kumbhakar et al. (2007) |
| **対象課題** | C1, C4 |

| PRO | CON |
|-----|-----|
| 理論的に確立されている（1996年〜） | 次元の呪い（高次元入力で困難） |
| 局所MLEで分布パラメータの異質性を許容 | バンド幅選択が結果に大きく影響 |
| 経済学コミュニティで受容されやすい | 計算負荷（各点で局所尤度を最適化） |
| NumPy/scipy のみで実装可能 | NN/GPほどの柔軟性はない |

**実装工数:** 中（2-3週間）
**研究上の新規性:** 低（手法は確立済み、Python実装の価値はある）

---

### 2.5 StoNED/CNLS型

| 項目 | 内容 |
|------|------|
| **概要** | 凸ノンパラメトリック最小二乗法 + SFA分解 |
| **代表文献** | Kuosmanen & Kortelainen (2012) |
| **対象課題** | C1, C5 |

| PRO | CON |
|-----|-----|
| 経済学的shape制約（単調性+凹性）が自然に組込み | pyStoNED が既に存在（差別化が必要） |
| DEA と SFA の橋渡し | CNLS は大規模で計算困難 |
| 関数形仮定なし | Stage 2 の分布仮定は残る |

**実装工数:** 中
**研究上の新規性:** 低（pyStoNED と重複）→ **候補から除外**

---

### 2.6 ツリーベース（EAT / 勾配ブースティング）

| 項目 | 内容 |
|------|------|
| **概要** | 形状制約付き決定木 or ブースティングでフロンティア推定 |
| **代表文献** | Esteve et al. (2020), Guillen et al. (2023) |
| **対象課題** | C1, C4 |

| PRO | CON |
|-----|-----|
| 高い解釈性（変数重要度、木構造） | **SFA誤差分解がない（確定論的）** |
| 大規模データに対応 | ノイズと非効率性を区別できない |
| 離散・カテゴリ変数に強い | SFAの枠組みから逸脱 |

**→ SFA誤差分解を持たないため、本プロジェクトの主目的に合致しない。候補から除外。**

---

### 2.7 BART-SFM（ベイズ加法回帰木型）

| 項目 | 内容 |
|------|------|
| **概要** | BART でフロンティアをモデル化 + ベイズ推論で誤差分解 |
| **代表文献** | Ferrara & Vidoli (2024) |
| **対象課題** | C1, C2, C4, C5 |

| PRO | CON |
|-----|-----|
| ノンパラメトリック + 不確実性定量化 + shape制約 | MCMC が遅い |
| 変数選択・交互作用検出が自然 | PyMC-BART への依存 |
| 部分依存プロットで解釈性を確保 | 先行実装なし |
| 多くの課題に同時対処 | 実装の複雑さ |

**実装工数:** 大（4-6週間）
**研究上の新規性:** 高

## 3. 比較マトリクス

### 3.1 課題への対応度（総括）

| モデル | C1 関数形 | C2 分布 | C3 識別 | C4 非線形 | C5 解釈性 | C6 パネル | C7 少量データ | 合計 |
|--------|:---------:|:-------:|:-------:|:---------:|:---------:|:---------:|:------------:|:----:|
| Parametric | - | - | - | - | ◎ | △ | ◎ | 6 |
| 2段階NN | ◎ | - | △ | ◎ | △ | ○ | △ | 9 |
| 同時推定NN | ◎ | ○ | ○ | ◎ | △ | ○ | - | 10 |
| 構造化NN | ◎ | - | ○ | ○ | ◎ | △ | △ | 9 |
| GP | ◎ | ○ | △ | ◎ | ○ | - | ◎ | 10 |
| Kernel | ○ | ○ | △ | ○ | ○ | - | ○ | 7 |
| BART-SFM | ◎ | ○ | △ | ◎ | ◎ | - | ◎ | 11 |
| StoNED/CNLS | ◎ | - | △ | ○ | ◎ | - | ○ | 9 |
| ツリーベース | ◎ | - | - | ◎ | ◎ | - | ○ | 8 |

`◎`=3点(強く対処) `○`=2点(部分的に対処) `△`=1点(限定的) `-`=0点(対処なし)

### 3.1.1 課題別の詳細評価

#### C1: フロンティア関数形への依存

> Cobb-Douglas / translog 等のパラメトリック関数形を事前に選択する必要があり、
> 選択によって効率性推定値が大きく変わる。

| モデル | 評価 | 根拠 |
|--------|:----:|------|
| Parametric | - | 関数形を事前指定する必要があり、この課題そのものを抱える |
| 2段階NN | ◎ | MLP が任意の連続関数を近似可能。関数形の事前指定不要 |
| 同時推定NN | ◎ | 同上。フロンティアと誤差分解を同時最適化するためより整合的 |
| 構造化NN | ◎ | サブネットが非線形フロンティアを学習。アーキテクチャ制約はあるが関数形は不要 |
| GP | ◎ | カーネル関数で滑らかさのみ指定。関数形は事前指定不要 |
| Kernel | ○ | 局所回帰で関数形を緩和するが、カーネル・バンド幅の選択に依存 |
| BART-SFM | ◎ | 回帰木アンサンブルが任意の関数を近似。関数形の仮定なし |
| StoNED/CNLS | ◎ | 凸性・単調性のshape制約のみ。関数形の仮定なし |
| ツリーベース | ◎ | 決定木/ブースティングが任意のフロンティアを近似 |

#### C2: 非効率性分布仮定への感度

> 非効率性 u の分布（half-normal, truncated-normal 等）の選択が
> 効率性推定値とランキングに影響する。

| モデル | 評価 | 根拠 |
|--------|:----:|------|
| Parametric | - | 特定の分布（half-normal 等）を仮定。選択に対して感度が高い |
| 2段階NN | - | Stage 2 でパラメトリック分布を仮定するため、同じ感度問題を持つ |
| 同時推定NN | ○ | 同時最適化により分布パラメータがフロンティアと共に調整される。感度は残るが緩和 |
| 構造化NN | - | 非効率性サブネットの出力分布は仮定が必要。構造は改善するが分布仮定は残る |
| GP | ○ | ベイズ的枠組みで事前分布の不確実性を吸収。分布仮定の影響を周辺化で緩和 |
| Kernel | ○ | 局所MLEで分布パラメータが共変量空間で変動可能。グローバルな仮定よりは柔軟 |
| BART-SFM | ○ | ベイズ推論で分布パラメータの事後分布を得る。分布は仮定するが不確実性を定量化 |
| StoNED/CNLS | - | Stage 2 の分解は分布仮定（半正規 or 指数）に依存 |
| ツリーベース | - | 確定論的手法のため非効率性分布を扱わない（N/A だが課題を回避ではなく無視） |

#### C3: 誤差分解の識別困難性

> 合成誤差 ε = v - u をノイズ v と非効率性 u に分離することの困難さ。
> SFA の最も根本的な課題。

| モデル | 評価 | 根拠 |
|--------|:----:|------|
| Parametric | - | 分布仮定のみに依存する脆弱な識別 |
| 2段階NN | △ | 柔軟なNNフロンティアが u を吸収するリスク。Shape制約で緩和可能だが根本解決ではない |
| 同時推定NN | ○ | フロンティアと誤差成分を同時推定するため、u 吸収問題を構造的に軽減。最も原理的なNN手法 |
| 構造化NN | ○ | アーキテクチャがフロンティア/非効率性/ノイズを明示分離。構造的な識別補助 |
| GP | △ | GPの柔軟性が u を吸収するリスクあり。事前分布による緩やかな正則化 |
| Kernel | △ | 局所MLEで各点の分解パラメータを推定するが、局所的に識別が不安定になり得る |
| BART-SFM | △ | BARTの柔軟性が u を吸収するリスク。ベイズ正則化で緩和するが根本解決ではない |
| StoNED/CNLS | △ | 凸性制約がフロンティアの過適合を防ぎ識別を助けるが、Stage 2 は分布仮定に依存 |
| ツリーベース | - | 誤差分解を行わない。ノイズと非効率性を区別しない |

#### C4: 非線形関係の捕捉不足

> 投入・産出間の複雑な非線形パターンを捉えられない問題。

| モデル | 評価 | 根拠 |
|--------|:----:|------|
| Parametric | - | Cobb-Douglas は対数線形、translog は2次項まで。複雑な非線形は不可 |
| 2段階NN | ◎ | 万能近似定理により任意の非線形を学習可能 |
| 同時推定NN | ◎ | 同上 |
| 構造化NN | ○ | 単調性制約が表現力を一部制限するが、各サブネットは非線形を学習可能 |
| GP | ◎ | カーネル関数の選択でさまざまな非線形パターンを捕捉 |
| Kernel | ○ | 局所回帰で非線形を近似するが、高次元では次元の呪いにより性能低下 |
| BART-SFM | ◎ | 回帰木アンサンブルが複雑な交互作用・非線形を自動検出 |
| StoNED/CNLS | ○ | 区分線形近似。凸性制約内では柔軟だが、非凸な関係は捉えられない |
| ツリーベース | ◎ | 決定木が任意の非線形パターンを表現可能 |

#### C5: 経済学的解釈性の欠如

> MLモデル導入時に生じるブラックボックス問題。
> 単調性（投入↑→産出↑）や凹性（限界生産力逓減）の経済学的性質が保証されない。

| モデル | 評価 | 根拠 |
|--------|:----:|------|
| Parametric | ◎ | 係数の経済学的解釈が直接的。弾力性 = 係数値 (Cobb-Douglas)。最も高い解釈性 |
| 2段階NN | △ | NNはブラックボックス。単調性はペナルティで近似的に強制可能だが保証なし。PDP等で事後的に解釈 |
| 同時推定NN | △ | 同上。フロンティアと誤差の同時推定で構造的にはやや改善するが、NN部分の解釈は困難 |
| 構造化NN | ◎ | アーキテクチャがSFA構造を反映。各サブネットの役割が明確。単調性をアーキテクチャで保証 |
| GP | ○ | カーネルの特性が解釈可能（長さスケール = 影響範囲）。事後分布で不確実性を可視化。ただし直接的な係数解釈はない |
| Kernel | ○ | 局所パラメータが各点での弾力性を提供。パラメトリックほど直接的ではないが、局所的解釈は可能 |
| BART-SFM | ◎ | 変数重要度、部分依存プロット、交互作用検出が自然。単調性制約 (monBART) あり |
| StoNED/CNLS | ◎ | 単調性・凹性が制約として組み込まれる。経済学的公理を満たすことが保証される |
| ツリーベース | ◎ | 木構造が直接的に解釈可能。変数重要度、分割ルールが明確。EATは単調性を保証 |

#### C6: パネルデータ・異質性への対応

> 同一企業の時系列観測（パネルデータ）の活用、
> 企業間の技術的異質性（heterogeneity）の扱い。

| モデル | 評価 | 根拠 |
|--------|:----:|------|
| Parametric | △ | 固定効果・ランダム効果SFAは確立されているが、本プロジェクトのスコープでは将来実装 |
| 2段階NN | ○ | Stage 1 のNNにパネル構造（企業ダミー、時間特徴量）を組込み可能。Kutlu (2024) で実証 |
| 同時推定NN | ○ | パネル構造をNN入力に反映可能。潜在変数で企業異質性をモデル化する拡張が自然 |
| 構造化NN | △ | パネル構造の明示的な組込みは追加設計が必要。先行研究でのパネル適用なし |
| GP | - | GPの計算量 O(N³) がパネルデータの大規模化で致命的。パネルGP-SFAの先行研究なし |
| Kernel | - | 各点の局所推定がパネル構造を活用しない。時間次元の追加で次元の呪いが悪化 |
| BART-SFM | - | BART自体はパネルデータに対応しにくい。先行研究でのパネル適用なし |
| StoNED/CNLS | - | 横断面データ向けに設計。パネル拡張は理論的に未整備 |
| ツリーベース | - | 横断面データ向けに設計。パネル構造の明示的な活用なし |

#### C7: 必要データ量とサンプル効率

> モデルが安定した効率性推定を返すために必要な観測数。
> SFA の応用分野（農業、銀行、公益事業等）ではデータが数十〜数百件と少ないことが多い。
> データ量はモデル選定の実務的な制約として極めて重要。

| モデル | 最小目安 | 推奨範囲 | 上限制約 | 根拠 |
|--------|:--------:|:--------:|:--------:|------|
| Parametric | **30〜50** | 50〜∞ | なし | 推定パラメータ数が少ない（Cobb-Douglas で入力数+3程度）。少量データで安定。MLEの漸近性は N>30 程度で十分 |
| 2段階NN | **300〜500** | 500〜10万 | なし | Stage 1 のMLP学習にはパラメータ数の10倍以上が目安。隠れ層 [64,32] で約3,000パラメータ → N>300 が実用下限 |
| 同時推定NN | **500〜1,000** | 1,000〜10万 | なし | フロンティアNNと誤差パラメータの同時推定はより多くのデータを要求。非凸最適化の安定にも量が必要 |
| 構造化NN | **300〜500** | 500〜5万 | なし | 3サブネットに分割されるため個々のパラメータは少ないが、全体の学習にはNNと同等のデータが必要 |
| GP | **50〜200** | 100〜5,000 | **~10,000** | 少量データで強力（事前分布が正則化として機能）。O(N³) の計算量で N>10,000 は変分近似が必須 |
| Kernel | **100〜200** | 200〜5,000 | **~10,000** | 局所推定のため各近傍に十分な点が必要。高次元では次元の呪いにより必要量が指数的に増加 |
| BART-SFM | **100〜200** | 200〜5,000 | **~50,000** | ツリーアンサンブルは少量データでもロバスト。事前分布が過適合を防止。MCMCの計算量が大規模の壁 |
| StoNED/CNLS | **50〜100** | 100〜2,000 | **~5,000** | 凸最適化で N 個の制約 → 計算量が N² に増加。大規模では実行不能 |
| ツリーベース | **100〜200** | 200〜∞ | なし | 単一木は少量で過適合リスク、ブースティングは中規模以上で安定。大規模に対してスケーラブル |

## 4. 選定の軸

モデル選定にあたり、以下の軸で優先順位を決める必要がある。

### 軸 A: 研究目的（学術貢献重視）

**優先:** 新規性 > 課題カバー率 > 実装容易性

推奨ポートフォリオ:
1. Parametric（必須ベースライン）
2. 同時推定NN or GP（高リスク・高リターン枠）
3. 2段階NN or BART-SFM（中リスク枠）

### 軸 B: ライブラリ目的（実用性重視）

**優先:** 課題カバー率 > 実装容易性 > 新規性

推奨ポートフォリオ:
1. Parametric（必須ベースライン）
2. 2段階NN（最も実用的なML-SFA）
3. Kernel（理論確立済み、安定）

### 軸 C: 探索目的（可能性の調査）

**優先:** 新規性 > 技術的面白さ > 安定性

推奨ポートフォリオ:
1. Parametric（ベースライン）
2. GP（未開拓領域）
3. 構造化NN（新しいアーキテクチャ設計）

## 5. 推奨案

### コアセット（いずれの方向性でも必須）

| 順序 | モデル | 理由 |
|:----:|--------|------|
| 1 | **ParametricSFA** | 全モデルのベンチマーク。ここが正しく動かなければ比較不能 |
| 2 | **2段階NN-SFA** | 最も実装しやすいML拡張。失敗リスクが低く、ML vs Parametric の比較が可能 |

### 拡張候補（方向性に応じて選択）

| モデル | 推奨する場合 | 推奨しない場合 |
|--------|-------------|---------------|
| **同時推定NN** | 2段階との比較で研究論文を書きたい場合 | 時間制約が厳しい場合 |
| **GP** | 未踏領域への挑戦が目的の場合 | 安定した成果が必要な場合 |
| **BART-SFM** | ベイズ推論・不確実性が重要な場合 | PyMC依存を避けたい場合 |
| **Kernel** | 確立手法のPython実装が目的の場合 | 新規性を重視する場合 |
| **構造化NN** | 解釈性とML柔軟性の両立が目的の場合 | 先行研究の裏付けを重視する場合 |
